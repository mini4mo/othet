1. Анализ предметной области

1.1 Описание предметной области

Управление задачами является важной частью повседневной жизни и профессиональной деятельности. Веб-приложение «Task Manager Pro» предназначено для упрощения процесса управления задачами, позволяя пользователям создавать, редактировать, удалять и отмечать задачи как выполненные. Приложение ориентировано на индивидуальных пользователей, таких как студенты, фрилансеры и офисные работники, которым требуется простой и интуитивно понятный инструмент для организации задач.

Основные требования к приложению:

  • Возможность добавления задачи с указанием текста и даты выполнения;
  • Отметка задач как выполненных;
  • Редактирование существующих задач;
  • Удаление задач;
  • Сохранение данных в локальном хранилище браузера.

1.2 Анализ существующих решений

На рынке существует множество приложений для управления задачами, от простых списков до комплексных систем управления проектами. Рассмотрим некоторые из них:

Todoist

Популярное приложение с возможностью создания проектов, подзадач и установки приоритетов. Имеет веб-версию и мобильные приложения.

Microsoft To Do

Простое приложение для управления задачами с интеграцией с другими продуктами Microsoft. Поддерживает напоминания и повторяющиеся задачи.

Trello

Визуальный инструмент управления проектами на основе канбан-досок. Позволяет организовывать задачи в виде карточек на досках.

Google Tasks

Встроенный в Google Calendar инструмент для управления задачами. Простой интерфейс с базовыми функциями.

Анализ существующих решений показал, что большинство из них предлагают избыточный функционал для базовых потребностей пользователей. Наше приложение «Task Manager Pro» фокусируется на простоте использования и основных функциях управления задачами, что делает его доступным для широкого круга пользователей.

1.3 Обоснование выбора технологий

Для разработки веб-приложения «Task Manager Pro» были выбраны следующие технологии:

HTML5

Современный стандарт разметки веб-страниц, обеспечивающий семантическую структуру документа и доступность.

CSS3

Язык стилей, позволяющий создать современный и адаптивный интерфейс с анимациями и переходами.

JavaScript (ES6+)

Язык программирования, обеспечивающий интерактивность и функциональность приложения на стороне клиента.

localStorage API

Web API для хранения данных в браузере пользователя без необходимости серверной части.

Выбор данных технологий обусловлен следующими факторами:

  • Отсутствие необходимости в серверной части для базовой функциональности;
  • Широкая поддержка браузерами;
  • Возможность создания адаптивного и интерактивного интерфейса;
  • Простота разработки и поддержки.

2. Разработка веб-приложения

2.1 Архитектура приложения

Приложение «Task Manager Pro» построено на модульной архитектуре, которая разделяет функциональность на несколько независимых компонентов:

  1. UI — отвечает за отображение интерфейса и обработку пользовательских действий.
  2. TaskManager — управляет задачами (добавление, редактирование, удаление, отметка как выполненные).
  3. Storage — обеспечивает асинхронное сохранение и загрузку задач в локальном хранилище.
  4. EventBus — реализует событийно-ориентированную модель для взаимодействия компонентов.

Модульная структура повышает читаемость кода, упрощает тестирование и поддержку.

2.2 Описание интерфейса

Интерфейс приложения включает следующие элементы:

  • Поле ввода для текста задачи;
  • Поле для выбора даты выполнения;
  • Кнопка добавления задачи;
  • Список задач с возможностью отметки, редактирования и удаления.

2.3 Реализация функциональности

Функциональность приложения реализована следующим образом:

  1. Добавление задачи: Пользователь вводит текст и, при необходимости, дату выполнения. Данные валидируются, после чего задача сохраняется в локальном хранилище.
  2. Редактирование задачи: При нажатии на кнопку «Редактировать» задача переходит в режим редактирования, где пользователь может изменить текст и дату.
  3. Удаление задачи: Удаление выполняется по нажатию кнопки «Удалить», после чего данные обновляются.
  4. Отметка выполнения: При нажатии на чекбокс задача отмечается как выполненная, меняется её визуальное представление.
  5. Сохранение данных: Все изменения автоматически сохраняются в localStorage браузера.

  • Корректности добавления, редактирования и удаления задач;

  • Обработки ошибок при пустом тексте задачи;
  • Адаптивности интерфейса на разных разрешениях экрана.
